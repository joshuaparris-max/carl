# Visual Art Direction

## Target Look

Dark fantasy dungeon with absurdist broadcast-game-show highlights.

The game should look grimy and dangerous, but the System UI should feel garish, artificial, and amused by the danger.

## Resolution and Scale

Recommended base:

- 16x16 tiles.
- 3x or 4x pixel scaling.
- Camera centered on a room or local area.
- UI rendered at native browser resolution, not pixel-scaled if readability suffers.

## Palette

Limit the first art pass to roughly 32 colors.

Core groups:

- Stone black.
- Warm dark gray.
- Cold blue-gray.
- Amber fire/System color.
- Teal Donut/companion color.
- Red danger/blood/debuff.
- Soft off-white text.
- Muted green healing/poison.

## Character Sprites

### Carl

Minimum animations:

- Idle.
- 4-direction walk, 4 frames each.
- Attack/lunge.
- Hit reaction.
- Death/down state.

Visual requirements:

- Leather jacket readable at 16x16.
- No-pants state readable in silhouette.
- Slightly awkward but determined posture.

### Princess Donut

Minimum animations:

- Idle.
- 4-direction walk, 4 frames each.
- Proud idle/head lift.
- Swipe/support action.
- Hit reaction.
- Hide/crouch state.

Visual requirements:

- Fluffy Persian silhouette.
- Teal or pale highlight for readability.
- Crown/show-cat energy without over-detailing.

### Mordecai

Minimum animations:

- Stressed idle.
- Talk animation.
- Frantic gesture.
- Desk-facing idle.

Visual requirements:

- Chupacabra-inspired shape if licensed/verified.
- Stressed posture.
- Silhouette different from enemies.

### Enemies

First three enemy types:

- Training goblin: simple, twitchy, weak.
- Patrol kobold: faster, patrol behavior.
- Sleeping crawler-turned-enemy: stationary until triggered.

Each needs:

- Idle.
- Move.
- Attack.
- Hit.
- Death.

## Tileset

First tiles:

- Floor.
- Wall.
- Corner wall.
- Desk.
- Chest.
- Shop/kiosk.
- Trap.
- Rough terrain.
- Filth/water hazard.
- Saferoom door.
- Locked saferoom door.
- Stair/exit.

Animated tiles:

- Chest glow.
- Hazard shimmer.
- Torch/fire.
- Door lock pulse.

## UI Art

UI should use:

- Dark panels.
- Amber trim.
- Teal companion accents.
- Red danger badges.
- Pixel-art icons for items.
- Large readable HP bars.
- Quest card states.

Avoid:

- Generic fantasy parchment.
- Overly clean sci-fi UI.
- Purple-heavy palette.
- Terminal emulator styling for the final game.

## Placeholder Art Strategy

Start with free placeholder tiles before drawing custom assets.

Recommended placeholder sources:

- Kenney.nl tiles and UI packs.
- Itch.io free dungeon tiles.
- Temporary colored squares if licensing is unclear.

Rules:

- Placeholder art must be clearly marked as placeholder.
- Do not polish placeholder art for too long.
- Replace main characters before public demo screenshots.
